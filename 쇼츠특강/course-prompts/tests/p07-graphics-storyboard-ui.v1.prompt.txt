P06에서 만든 그래픽형 쇼츠 기획안을 바탕으로, 그래픽형 storyboard JSON과 대본/자막 검수 흐름을 만들어줘.

중요:
- 아직 Typecast TTS 호출은 하지 마.
- 아직 영상 렌더링도 하지 마.
- 이번 단계는 "그래픽형 storyboard JSON + 검수 UI 연결"까지만 한다.
- 현재 폴더 안에서만 작업해줘.
- API 키를 출력하거나 파일에 하드코딩하지 마.
- 한글 JSON은 PowerShell Get-Content로 읽지 말고, Node.js fs.readFileSync(..., "utf8") + JSON.parse 또는 프롬프트 본문 기준으로 처리해줘.

먼저 아래 파일을 읽어 P06 기획안을 확인해줘.
- course-prompts/tests/p06-graphics-plan.v1.result.txt

해야 할 일:
1. 그래픽형 storyboard JSON을 만들어줘.
   - 저장 위치: storyboards/graphics-cycloid.storyboard.json
   - projectName: graphics-cycloid
   - 1080x1920, 30fps 기준
   - 각 scene에는 id, type="graphics", durationSeconds, headline, caption, narration, subtitleChunks를 포함해줘.
   - 각 scene에는 Remotion 구현을 위한 graphics 객체를 포함해줘.
     예: mode, paths, labels, balls, timers, highlightColor, 설명에 필요한 파라미터 등
   - subtitleChunks는 2~5어절 기준으로 자연스럽게 나눠줘.
   - chunk를 이어 붙이면 narration과 정확히 같아야 해.
   - 중복/누락 금지.
   - 조사, 복합서술어, 관형어+명사가 어색하게 찢어지지 않게 해.

2. 기존 caption review UI를 그래픽형 storyboard에도 사용할 수 있게 연결해줘.
   - 기존 tools/caption-review-server.mjs가 --storyboard 인자를 이미 지원하면 재사용해도 된다.
   - package.json에 그래픽형 검수 실행 스크립트를 추가해줘.
     예: "caption:review:graphics": "node tools/caption-review-server.mjs --storyboard storyboards/graphics-cycloid.storyboard.json --port 8788"

3. 수업용으로 보기 쉬운 보조 문서를 만들어줘.
   - assets/graphics-cycloid/graphics-storyboard.md
   - 씬별 대본, 자막 청크, 그래픽 구현 요약을 정리해줘.

검증:
- JSON 파싱
- 모든 scene에서 chunks를 이어 붙인 문장이 narration과 정확히 같은지 확인
- npm run typecheck
- package.json script 확인

마지막 응답에는 만든/수정한 파일, 실행 방법, 검증 결과만 정리해줘.
