세팅

세팅 프롬프트

프롬프트그대로 붙여넣으세요
Codex에서 AI 쇼츠 제작용 작업환경을 세팅해줘. 작업 폴더를 하나 만들고, Remotion을 설치해줘. Typecast API 키를 환경변수로 연결할 수 있게 구성해줘. 이후 씬 단위로 TTS 음성을 생성할 수 있게 스크립트를 준비해줘. 그리고 앞으로 쇼츠를 만들 때 사용할 수 있도록 이미지 생성형 쇼츠와 Remotion 그래픽형 쇼츠를 모두 처리할 수 있는 구조를 만들어줘. 이미지 생성형은 이미지, TTS, 자막, 전환효과, 효과음을 병합하는 방식이고, 그래픽형은 숫자, 선, 도형, 그래프 같은 시각요소를 Remotion으로 애니메이션화하는 방식이야.
※ 강의장 안내 — 만약 Typecast API 호출이 안 될 경우 Warp CLI를 설치해 우회하세요. 강의장 내에서 동시에 호출하면 같은 IP의 동시 호출로 감지될 수 있습니다. (문제가 없다면 그대로 호출)
※ 스펙 참고 — Typecast API/CLI 스펙은 typecast.ai/docs/ko/cli-reference/overview 를 참고할 것.

수학, 과학, 경제, 데이터 중에서 쇼츠로 만들기 좋은 흥미로운 지식 하나를 조사해서 골라줘. 아직 Remotion 제작이나 렌더링은 하지 말고, 먼저 기획안만 표로 정리해줘. 표에는 다음 열을 넣어줘. · 장면번호 · 장면묘사 · 실제 대본문장 · 예상 시간초 장면묘사는 하나마나 한 설명이 아니라, 화면 중앙에 어떤 숫자·선·도형·그래프·게이지가 등장하고 어떤 순서로 움직이는지 구체적으로 써줘. 대본문장은 실제 TTS로 읽을 문장 기준으로 자연스럽게 써줘. 내가 승인하면 그다음 단계로 넘어갈게.

프롬프트복사해서 입력
웹 UI에서 저장한 JSON을 기준으로 Remotion 그래픽형 쇼츠를 만들어줘. 씬마다 파일을 나누어 모듈식으로 작성하고, 각 씬은 JSON의 장면묘사를 기준으로 숫자·선·도형·그래프·게이지·자막으로 구성해줘. 각 씬의 실제 대본문장을 Typecast로 TTS 생성하고, 생성된 음성 길이에 맞춰 씬 duration을 조정해줘. 자막은 저장된 자막 청크를 사용하고, 화면 중앙 시각요소 아래에 쇼츠 자막처럼 크게 배치해줘. 상단 헤더나 하단 푸터는 넣지 말고, 메인 시각요소와 자막 중심으로 구성해줘. 최종 결과는 세로 쇼츠 mp4로 렌더링해줘.
