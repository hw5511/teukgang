# 배포 및 인증 실험 기록

## 최종 결론

수업에서는 Cloudflare Pages + Wrangler CLI를 사용한다.

가장 중요한 이유는 초보자에게 인증과 배포가 단순하기 때문이다.

```text
Codex가 wrangler login 실행
→ 브라우저가 열림
→ 수강생이 Cloudflare 로그인 승인
→ Codex가 Pages 배포
→ pages.dev 링크 확인
```

## Cloudflare Pages

테스트 결과:

- `npx -y wrangler login` 실행 시 브라우저 OAuth 창이 바로 열렸다.
- 로그인 승인 후 Wrangler CLI로 Pages 배포가 가능했다.
- 최종 테스트 게임 URL: https://codex-iron-rescue.pages.dev

사용했던 흐름:

```powershell
npx -y wrangler login
npx -y wrangler pages project create codex-iron-rescue --production-branch main
npx -y wrangler pages deploy . --project-name codex-iron-rescue --commit-dirty=true
```

수업용 요청 문장:

```text
wrangler cli를 설치해줘.
그다음 wrangler login을 실행해줘.
브라우저가 열리면 내가 직접 Cloudflare 로그인을 승인할게.
인증이 완료되면 이 게임을 Cloudflare Pages로 배포해줘.
```

## GitHub CLI + GitHub Pages

테스트 및 논의 결과:

- `gh auth login --web --git-protocol https` 인증은 가능하다.
- 다만 수업용으로는 8자리 디바이스 코드 확인, 브라우저 인증, repo 생성, push, GitHub Pages 설정까지 단계가 길다.
- 인증 코드를 크게 보여주는 래핑 스크립트도 실험했지만 Windows/macOS 모두에서 안정적으로 설명하기에는 복잡했다.
- 따라서 초보자 대상 1시간 실습의 기본 배포 방식으로는 제외했다.

참고로 현재 GitHub CLI 상태:

- 계정: `hw5511`
- 연결된 레포: `https://github.com/hw5511/teukgang`

## Vercel

테스트 결과:

- 배포 자체는 가능했다.
- 테스트 URL: `https://metal-slug-game.vercel.app`
- 이후 삭제했고 404 상태를 확인했다.

수업 판단:

- 좋은 배포 도구지만, 이번 수업에서는 Cloudflare Pages가 더 단순하다.

## Netlify

테스트 결과:

- 배포 자체는 가능했다.
- 테스트 URL: `https://iron-rescue-codex.netlify.app`
- 이후 삭제했고 404 상태를 확인했다.

수업 판단:

- 가능하지만 계정/사이트 연결 흐름이 Cloudflare보다 수업용으로 덜 단순했다.

## Cloudflare Tunnel

판단:

- 임시 공유에는 적합하다.
- 하지만 수업 목표가 "공유 가능한 배포 URL 완성"이므로 Pages가 더 적합하다.

## 수업 중 주의할 점

- Cloudflare 가입 시간을 수업 순서에 반드시 포함한다.
- `wrangler login`은 Codex가 실행하되, 로그인 승인은 수강생이 직접 한다.
- OAuth 승인, 2FA, 비밀번호 입력은 Codex가 대신하지 않는다.
- 배포가 끝난 뒤에는 반드시 모바일에서도 `pages.dev` 링크를 열어 확인한다.
