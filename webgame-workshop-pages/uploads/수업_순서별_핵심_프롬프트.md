# Codex 웹게임 특강 순서별 핵심 프롬프트

이 문서는 `Codex로 나만의 웹 게임 만들기` 수업에서 수강생이 단계별로 복사해서 사용할 수 있는 프롬프트 모음이다.

수업의 목표는 코딩 문법 설명이 아니라, Codex에게 원하는 결과를 말하고, 실행 결과를 확인하고, 다시 피드백하는 제작 흐름을 경험하게 하는 것이다.

## 0. 수업 시작 전 안내

### 목적

- Codex가 현재 작업할 폴더를 정확히 인식하게 한다.
- 수강생이 다운로드 폴더 전체나 바탕화면 전체를 작업 공간으로 쓰지 않게 한다.
- 이후 생성되는 파일이 한 프로젝트 폴더 안에 모이게 한다.

### 핵심 프롬프트

```text
현재 폴더를 확인해줘.

이 폴더 안에 오늘 만들 웹 게임 프로젝트를 만들 거야.
HTML, CSS, JavaScript 파일을 나누어 관리할 수 있게 준비해줘.

아직 코드는 만들지 말고,
어떤 파일 구조로 만들면 좋을지 먼저 간단히 설명해줘.
```

### 막힐 때 추가 요청

```text
내가 지금 Codex에서 열고 있는 작업 폴더가 어디인지 알려줘.
그리고 오늘 수업용 웹 게임 프로젝트를 만들기에 적절한 위치인지 확인해줘.
```

## 1. 게임 콘셉트 정하기

### 목적

- 처음부터 구현을 시작하지 않고 만들 게임의 방향을 정한다.
- 원작 캐릭터나 상표권이 있는 캐릭터를 그대로 쓰지 않도록 안내한다.
- 장르, 분위기, 조작, 목표를 한 번에 정리한다.

### 핵심 프롬프트

```text
나는 Codex를 처음 써보는 수강생이고,
오늘 브라우저에서 실행되는 간단한 2D 웹 게임을 만들고 싶어.

게임 콘셉트를 같이 정리해줘.

조건:
- HTML/CSS/JavaScript로 만들 수 있는 간단한 게임
- 수업 시간 안에 만들 수 있을 정도로 범위가 작을 것
- 원작 캐릭터를 그대로 따라 하지 않는 오리지널 게임일 것
- 플레이어가 조작할 캐릭터, 적 또는 장애물, 점수 요소가 있을 것
- 마지막에 친구에게 URL로 공유할 수 있는 결과물이 될 것

먼저 만들기 좋은 게임 아이디어를 3개 제안해주고,
각 아이디어마다 장르, 조작 방법, 목표, 난이도를 짧게 비교해줘.
```

### 선택 후 정리 프롬프트

```text
좋아, 나는 [선택한 게임 아이디어]로 만들고 싶어.

이 게임의 핵심 콘셉트를 한 문단으로 정리해줘.
그리고 바로 다음 단계에서 HTML/CSS/JavaScript로 만들 수 있도록
필수 기능 목록과 나중에 추가하면 좋은 기능 목록을 나눠줘.
```

## 2. 첫 번째 게임 만들기

### 목적

- 완벽한 게임보다 먼저 실행되는 첫 버전을 만든다.
- 파일을 `index.html`, `style.css`, `script.js`처럼 이해하기 쉽게 나눈다.
- Codex가 로컬 테스트까지 이어갈 수 있게 요청한다.

### 핵심 프롬프트

```text
방금 정리한 콘셉트로 웹 게임 첫 버전을 만들어줘.

조건:
- HTML/CSS/JavaScript만 사용해줘.
- 브라우저에서 바로 실행되는 정적 웹 게임으로 만들어줘.
- 파일은 index.html, style.css, script.js로 나눠줘.
- 플레이어 이동 기능을 넣어줘.
- 점수 또는 아이템 획득 요소를 넣어줘.
- 적, 장애물, 또는 피해야 하는 요소를 넣어줘.
- 게임 시작 화면, 게임오버 화면, 다시 시작 기능을 넣어줘.
- 코드 구조는 초보자가 읽어도 이해하기 쉽게 작성해줘.

완성 후에는 Codex에서 바로 로컬 테스트할 수 있는 실행 방법도 알려줘.
가능하면 로컬 서버를 실행해서 내가 브라우저에서 확인할 수 있게 해줘.
```

### 단순화가 필요할 때

```text
기능이 너무 많아서 복잡하면,
먼저 작동하는 최소 버전으로 줄여줘.

이번 단계에서는 다음 기능만 꼭 넣어줘:
- 플레이어 이동
- 점수 획득
- 장애물 또는 적
- 게임오버
- 다시 시작

나머지 기능은 나중에 추가할 수 있게 코드 구조만 정리해줘.
```

## 3. 로컬 테스트

### 목적

- 수강생이 직접 실행 결과를 확인하게 한다.
- 문제를 “에러가 났어요”가 아니라 구체적인 상태로 설명하게 한다.
- Codex가 로컬 서버 실행, 브라우저 확인, 오류 수정까지 이어가게 한다.

### 핵심 프롬프트

```text
지금 만든 게임을 로컬에서 실행해서 테스트할 수 있게 해줘.

필요하면 간단한 로컬 서버를 실행해줘.
브라우저에서 열 주소를 알려주고,
내가 확인해야 할 테스트 항목도 체크리스트로 정리해줘.

테스트 항목에는 다음을 포함해줘:
- 게임 화면이 정상적으로 보이는지
- 시작 버튼이 작동하는지
- 플레이어 조작이 되는지
- 점수나 아이템이 작동하는지
- 적 또는 장애물과 충돌했을 때 게임오버가 되는지
- 다시 시작이 되는지
```

### 버그 전달 프롬프트

```text
테스트해보니 문제가 있어.

문제 상황:
- [예: 시작 버튼을 눌러도 게임이 시작되지 않음]
- [예: 점프가 되지 않음]
- [예: 적과 부딪혀도 게임오버가 되지 않음]

이 문제를 원인부터 확인하고 수정해줘.
수정한 뒤 다시 로컬에서 확인할 수 있게 해줘.
```

## 4. 플레이 감각 다듬기

### 목적

- 수강생이 직접 플레이해보고 자연어 피드백을 주게 한다.
- 점프감, 속도, 난이도, 피드백 효과를 빠르게 개선한다.

### 핵심 프롬프트

```text
게임을 직접 해보니 아직 플레이 감각이 조금 어색해.
전체적으로 더 재미있고 조작하기 쉽게 다듬어줘.

중점적으로 봐줄 부분:
- 플레이어 이동 속도
- 점프 높이 또는 반응성
- 적이나 장애물의 등장 속도
- 아이템을 먹었을 때 피드백
- 점수를 얻는 느낌
- 게임오버가 너무 갑작스럽지 않은지

초보자가 플레이해도 너무 어렵지 않게 조정해줘.
수정 후 어떤 값을 어떻게 바꿨는지도 간단히 설명해줘.
```

### 구체적 피드백 예시

```text
점프가 너무 낮아서 답답해.
플레이어가 조금 더 높게 점프하고,
착지감이 자연스럽게 느껴지도록 조정해줘.
```

```text
적이 너무 빨리 나와서 처음부터 어렵게 느껴져.
초반에는 쉽게 시작하고,
시간이 지날수록 조금씩 어려워지게 만들어줘.
```

```text
아이템을 먹었는지 잘 모르겠어.
아이템을 획득하면 점수 변화, 짧은 효과, 화면 메시지 중 하나 이상으로
확실히 알 수 있게 만들어줘.
```

## 5. 이미지 에셋 만들기

### 목적

- 게임을 도형 기반에서 캐릭터/아이템 이미지 기반으로 발전시킨다.
- 배경 제거 또는 투명 PNG가 필요하다는 점을 명확히 한다.
- 한 장에 모든 것을 넣기보다 개별 에셋으로 요청한다.

### 핵심 프롬프트

```text
이 게임에 사용할 이미지 에셋을 만들어줘.

필요한 에셋:
- 플레이어 캐릭터
- 적 캐릭터 또는 장애물
- 점수 아이템
- 회복 아이템 또는 보너스 아이템
- 목표 지점 또는 클리어 오브젝트

스타일:
- 귀엽고 밝은 2D 게임 느낌
- 간단한 도트 그래픽 또는 픽셀 아트 느낌
- 작은 화면에서도 알아보기 쉬운 실루엣
- 같은 게임에 들어가도 어색하지 않게 통일된 색감

게임 코드에 바로 적용하기 쉽게
각 에셋을 따로 사용할 수 있는 투명 배경 PNG 형태로 준비해줘.
만약 완전한 투명 배경이 어렵다면,
배경 제거가 쉬운 단색 배경으로 만든 뒤 투명화할 수 있게 해줘.
```

### 에셋 저장 요청

```text
만든 에셋을 게임 프로젝트 안의 assets 폴더에 저장해줘.

파일 이름은 역할을 알 수 있게 다음처럼 정리해줘:
- player.png
- enemy.png
- coin.png
- heart.png
- goal.png

저장 후 실제 파일 경로를 알려줘.
```

## 6. 이미지 에셋 적용

### 목적

- 기존 도형 그래픽을 이미지 기반으로 교체한다.
- 이미지가 깨지거나 경로가 틀렸을 때 확인할 수 있게 한다.

### 핵심 프롬프트

```text
방금 만든 PNG 에셋들을 게임에 적용해줘.

적용할 내용:
- 플레이어 도형을 player.png 이미지로 교체
- 적 또는 장애물을 enemy.png 이미지로 교체
- 점수 아이템을 coin.png 이미지로 교체
- 회복 아이템이 있다면 heart.png 이미지로 교체
- 목표 지점이 있다면 goal.png 이미지로 교체

주의할 점:
- 이미지 파일 경로가 정확해야 해.
- 이미지가 로드되지 않을 경우에도 게임이 완전히 멈추지 않게 해줘.
- 기존 충돌 판정은 유지하되, 이미지 크기에 맞게 보기 좋게 조정해줘.

적용 후 로컬에서 이미지가 정상적으로 보이는지 확인할 수 있게 해줘.
```

### 이미지가 안 보일 때

```text
게임 화면에서 이미지가 보이지 않아.

다음을 확인해서 고쳐줘:
- assets 폴더 위치
- 이미지 파일 이름
- HTML 또는 JavaScript에서 참조하는 경로
- 이미지 로딩 시점
- 브라우저 콘솔 오류

수정 후 어떤 경로로 이미지를 불러오는지 설명해줘.
```

## 7. 게임 완성도 높이기

### 목적

- 결과물이 “예제 코드”가 아니라 “작은 게임”처럼 보이게 한다.
- 시작/진행/실패/성공 흐름을 정리한다.
- 수업 시간 안에서 효과가 큰 기능을 우선한다.

### 핵심 프롬프트

```text
이 게임의 완성도를 높여줘.

우선순위:
1. 시작 화면을 더 명확하게 만들기
2. 게임 중 점수, 체력, 남은 시간 또는 진행도를 잘 보이게 하기
3. 아이템 획득 효과 추가하기
4. 적이나 장애물에 부딪혔을 때 피격 효과 추가하기
5. 게임오버 화면을 보기 좋게 만들기
6. 클리어 조건과 클리어 화면 추가하기
7. 다시 시작 흐름을 자연스럽게 만들기

효과는 너무 복잡하지 않아도 괜찮아.
짧은 애니메이션, 색 변화, 화면 흔들림, 메시지 표시처럼
수업 시간 안에 구현 가능한 방식으로 해줘.

수정 후 어떤 기능이 추가되었는지 목록으로 정리해줘.
```

### 시간이 부족할 때

```text
수업 시간이 부족하니까 완성도 개선을 꼭 필요한 것만 해줘.

이번에는 다음 4가지만 우선 적용해줘:
- 시작 화면
- 점수 표시
- 게임오버 화면
- 다시 시작 버튼

복잡한 스테이지나 고급 애니메이션은 나중에 할 수 있게 남겨둬.
```

## 8. 모바일 반응형과 터치 조작

### 목적

- 배포 후 휴대폰에서 링크를 열어도 플레이할 수 있게 한다.
- 캔버스 또는 게임 화면이 화면 밖으로 넘치지 않게 한다.
- 모바일 전용 터치 버튼을 추가한다.

### 핵심 프롬프트

```text
이 게임을 모바일에서도 플레이할 수 있게 개선해줘.

필수 조건:
- 화면 크기에 맞게 게임 영역이 줄어들거나 조정될 것
- 휴대폰 화면에서 좌우 스크롤이 생기지 않을 것
- 모바일에서는 터치 버튼으로 조작할 수 있을 것
- 터치 버튼은 엄지로 누르기 쉽게 화면 아래쪽에 배치할 것
- 데스크톱에서는 기존 키보드 조작도 계속 작동할 것

터치 버튼:
- 왼쪽 이동
- 오른쪽 이동
- 점프
- 공격 또는 액션 버튼이 있다면 포함
- 다시 시작 버튼이 필요하면 모바일에서도 누르기 쉽게 표시

수정 후 데스크톱과 모바일에서 각각 어떻게 조작하는지 설명해줘.
```

### 모바일 확인 요청

```text
모바일 화면 기준으로 레이아웃을 다시 확인해줘.

확인할 것:
- 게임 화면이 너무 작거나 잘리지 않는지
- 터치 버튼이 게임 화면을 과하게 가리지 않는지
- 버튼 사이 간격이 충분한지
- 텍스트가 화면 밖으로 나가지 않는지
- 세로 화면과 가로 화면 중 어느 쪽에 더 적합한지

문제가 있으면 바로 수정해줘.
```

## 9. 배포 전 점검

### 목적

- 배포 전에 정적 파일만으로 실행 가능한지 확인한다.
- 불필요한 파일이나 로컬 전용 설정을 줄인다.
- 수강생이 배포 실패 시 원인을 이해할 수 있게 한다.

### 핵심 프롬프트

```text
Cloudflare Pages에 배포하기 전에 프로젝트를 점검해줘.

확인할 것:
- index.html이 루트에 있는지
- style.css와 script.js 경로가 올바른지
- assets 폴더 이미지 경로가 올바른지
- 브라우저 콘솔 오류가 없는지
- 로컬 서버에서 정상 실행되는지
- 배포에 필요 없는 임시 파일이 있는지

정적 웹사이트로 배포 가능한 상태인지 확인하고,
문제가 있으면 수정해줘.
```

## 10. Cloudflare Pages 배포

### 목적

- GitHub 없이도 공개 URL을 만든다.
- 로그인과 OAuth 승인은 수강생이 직접 한다.
- Codex는 Wrangler 실행과 배포 명령을 맡는다.

### 수업 전 안내 문장

```text
이제 만든 게임을 웹에 배포하기 위해 Cloudflare 계정이 필요합니다.
cloudflare.com에 가입하거나 로그인해 주세요.

비밀번호나 인증 코드는 Codex에게 알려주지 않습니다.
브라우저에 로그인 창이 뜨면 여러분이 직접 승인하면 됩니다.
```

### 핵심 프롬프트

```text
이 게임을 Cloudflare Pages로 배포해줘.

먼저 Wrangler CLI가 필요한지 확인하고,
필요하면 npx로 실행해줘.

그다음 wrangler login을 실행해줘.
브라우저가 열리면 내가 직접 Cloudflare 로그인을 승인할게.

인증이 완료되면 현재 프로젝트 폴더를 Cloudflare Pages로 배포해줘.
배포가 끝나면 접속 가능한 pages.dev URL을 알려줘.
```

### 프로젝트 이름까지 지정할 때

```text
Cloudflare Pages 프로젝트 이름은 [프로젝트이름]으로 해줘.

다음 흐름으로 진행해줘:
1. wrangler login 실행
2. 내가 브라우저에서 승인할 때까지 기다리기
3. Pages 프로젝트 생성
4. 현재 폴더 배포
5. 배포 URL 알려주기

배포 중 오류가 나면 마지막 오류 메시지를 기준으로 원인을 설명하고 수정해줘.
```

## 11. 배포 결과 확인

### 목적

- 배포 URL이 열리는 것만으로 끝내지 않는다.
- 실제 게임 화면, 조작, 이미지, 모바일 UI가 배포 환경에서도 동작하는지 확인한다.

### 핵심 프롬프트

```text
방금 배포한 pages.dev URL에서 게임이 정상 작동하는지 확인해줘.

확인할 것:
- 페이지가 열리는지
- 게임 화면이 보이는지
- 이미지 에셋이 깨지지 않는지
- 시작 버튼이 작동하는지
- 키보드 조작이 되는지
- 모바일 터치 UI가 보이는지
- 게임오버와 다시 시작이 되는지

배포 환경에서만 문제가 생기면 원인을 찾아서 수정하고 다시 배포해줘.
```

### 공유 전 마무리 요청

```text
최종 배포 URL을 수강생이 공유하기 좋게 정리해줘.

다음 형식으로 알려줘:
- 게임 제목
- 한 줄 소개
- 플레이 방법
- 배포 URL
- 모바일에서 확인할 점
```

## 12. 수업 중 문제가 생겼을 때

### 오류 메시지가 있을 때

```text
아래 오류 메시지를 보고 문제를 고쳐줘.

오류 메시지:
[여기에 오류 메시지 붙여넣기]

내가 코딩 초보자라서,
왜 문제가 생겼는지 짧게 설명하고
수정까지 진행해줘.
```

### 화면은 뜨지만 이상할 때

```text
게임 화면은 뜨지만 동작이 이상해.

현재 상태:
- [예: 캐릭터가 화면 밖으로 나감]
- [예: 버튼이 눌리지 않음]
- [예: 점수가 너무 빨리 올라감]

원인을 찾아서 수정해줘.
수정 후 다시 테스트할 수 있게 해줘.
```

### 시간이 부족할 때

```text
수업 시간이 부족해.

지금 상태에서 배포 가능한 최소 완성본으로 정리해줘.

우선순위:
1. 게임이 실행될 것
2. 시작과 다시 시작이 될 것
3. 점수 또는 목표가 있을 것
4. 모바일에서 최소한 플레이 가능할 것
5. Cloudflare Pages로 배포 가능할 것

복잡한 기능은 제거하거나 단순화해도 괜찮아.
```

## 13. 수업용 통합 프롬프트

### 목적

- 시간이 부족하거나 실습 속도 차이가 클 때 한 번에 진행한다.
- 전체 흐름을 Codex에게 맡기되, 로컬 테스트와 배포까지 포함한다.

### 통합 프롬프트

```text
나는 Codex를 처음 써보는 수강생이고,
AI와 함께 간단한 2D 웹 게임을 만들고 싶어.

HTML/CSS/JavaScript만 사용해서
브라우저에서 실행되는 오리지널 2D 웹 게임을 만들어줘.

게임 조건:
- 플레이어 이동 기능
- 점프 또는 액션 기능
- 적, 장애물, 또는 피해야 하는 요소
- 아이템 또는 점수 획득 요소
- 시작 화면
- 게임오버 화면
- 클리어 조건 또는 목표
- 다시 시작 기능

개발 조건:
- 파일은 index.html, style.css, script.js로 나눠줘.
- 코드 구조는 초보자가 이해하기 쉽게 작성해줘.
- 로컬에서 바로 테스트할 수 있게 해줘.
- 필요한 이미지 에셋은 투명 배경 PNG로 만들거나 적용하기 쉽게 준비해줘.
- 모바일에서도 플레이할 수 있게 반응형과 터치 조작 UI를 추가해줘.

진행 방식:
1. 먼저 최소 실행 버전을 만들어줘.
2. 로컬에서 실행해 확인해줘.
3. 문제가 있으면 수정해줘.
4. 에셋을 적용해줘.
5. 모바일 대응을 추가해줘.
6. 배포 전 점검을 해줘.
7. Cloudflare Pages로 배포해줘.

Cloudflare 로그인이 필요하면 wrangler login을 실행해줘.
브라우저가 열리면 내가 직접 로그인하고 승인할게.
인증이 완료되면 Pages로 배포하고 최종 URL을 알려줘.
```

이 게임에 스테이지를 3개 추가해줘.

이 게임에 보스전을 추가해줘.

친구들이 점수를 비교할 수 있는 랭킹 기능을 만들려면
